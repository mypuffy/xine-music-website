# JSJ Music - Hard Techno Label Website

Modern, futuristic website for JSJ Music, a Hard Techno record label. Built with Next.js and designed for static hosting with Caddy.

## Features

- Modern cyber-themed design with neon effects and glassmorphism
- Fully responsive layout
- Static site export for fast, efficient hosting
- Docker deployment with Caddy
- Optimized for performance
- No server-side dependencies
- SEO friendly

## Tech Stack

- **Next.js 14** - React framework with static export
- **TypeScript** - Type safety
- **Tailwind CSS** - Utility-first CSS framework
- **Lucide React** - Modern icon library
- **Caddy 2** - Modern web server with automatic HTTPS

## Production Deployment

### Option 1: Docker with Caddy (Recommended)

1. Build the static site:
```bash
npm run build
```

2. Start with Docker Compose:
```bash
docker-compose up -d
```

The site will be available at `http://localhost`

3. View logs:
```bash
docker-compose logs -f web
```

4. Stop:
```bash
docker-compose down
```

### Option 2: Automated Deployment Script

Use the deployment script for complete automated deployment:

```bash
chmod +x deploy.sh
./deploy.sh
```

This script will:
- Install dependencies with `npm ci`
- Build the static site
- Build and start Docker containers

### Option 3: Manual Caddy Setup

1. Build the site:
```bash
npm run build
```

2. Install Caddy (if not using Docker):
```bash
# macOS
brew install caddy

# Ubuntu/Debian
sudo apt install caddy

# Or download from https://caddyserver.com/download
```

3. Run Caddy:
```bash
caddy run --config Caddyfile
```

### Production Configuration with HTTPS

For production deployment with automatic HTTPS:

1. Copy the production Caddyfile example:
```bash
cp Caddyfile.production.example Caddyfile
```

2. Update the domain and email in `Caddyfile`:
```caddyfile
{
    email your-email@example.com
}

your-domain.com {
    # ... configuration
}
```

3. Update `docker-compose.yml` ports if needed (usually 443 for HTTPS)

4. Point your domain's DNS to your server

5. Start the containers:
```bash
docker-compose up -d
```

Caddy will automatically obtain and manage SSL certificates from Let's Encrypt.

## Docker Commands

```bash

docker-compose up -d

# Stop containers
docker-compose down

# View logs (follow mode)
docker-compose logs -f web

# Rebuild and restart
docker-compose up -d --build

# Check container status
docker-compose ps

# View resource usage
docker stats jsj-music-web
```

## Environment Variables

For local development, create a `.env.local` file:

```env
NEXT_PUBLIC_SITE_URL=http://localhost:3000
NEXT_PUBLIC_SITE_NAME=JSJ Music
```

Note: Environment variables must be prefixed with `NEXT_PUBLIC_` to be accessible in the browser.

## Customization

### Updating Content

Edit the following files to customize content:

- **Homepage**: `app/page.tsx`
- **Header/Navigation**: `components/layout/Header.tsx`
- **Footer**: `components/layout/Footer.tsx`
- **Global Styles**: `app/globals.css`
- **Theme Colors**: `tailwind.config.js`

### Styling

- **Color Scheme**: Update colors in `tailwind.config.js`
- **Animations**: Modify animations in `app/globals.css`
- **Layout**: Edit component files in `app/` and `components/`

### Images

Replace Pexels placeholder images with your own:
- Artist photos
- Album/release covers
- Background images

Images should be placed in the `public/` directory or served from a CDN.

## Performance Optimization

The site is optimized for performance:
- Static export (no server-side rendering overhead)
- Optimized images with `next/image` (when using `<Image />`)
- Gzip and Zstandard compression via Caddy
- Aggressive caching for static assets (1 year)
- Minimal JavaScript bundle
- CSS optimization via Tailwind's purge

## Troubleshooting

### Build Issues

If you encounter build errors:

```bash
# Clean install
rm -rf node_modules package-lock.json
npm install

# Clean build
rm -rf .next out
npm run build
```

### Docker Issues

```bash
# View container logs
docker-compose logs web

# Restart containers
docker-compose restart

# Rebuild from scratch
docker-compose down
docker-compose up -d --build
```

### Port Already in Use

If port 80 or 443 is already in use, update `docker-compose.yml`:

```yaml
ports:
  - "8080:80"  # Use port 8080 instead
```
